# media-core

`media-core` 是 VOS-RS 软交换平台的媒体面核心组件，作为协议与核心算法库，为信令边缘节点（`sip-edge`）和媒体边缘节点（`media-edge`）提供底层媒体包处理、转码、混音、录音及会话描述协议（SDP）重写支持。

---

## 模块介绍

本 Crate 包含以下核心功能模块：

### 1. SDP 解析与重写 (`sdp`)
* **快速重写机制**：实现高性能的 SDP 快速重写流，修改媒体连接地址和端口映射，避免完整的 SDP 解析开销。
* **媒体级覆盖**：支持 RFC 4566 规范中媒体级（`m=audio` 段）连接信息（`c=` 行）覆盖会话级连接信息的解析规则。
* **协商策略**：提供灵活的媒体协商验证（如 `AUDIO_ONLY` 策略，用于主动拒绝 T.38 等非音频提议）。

### 2. 会议混音与控制 (`conference`)
* **多方混音引擎**：实现多方音频流的混音算法，生成符合电话格式的 G.711 混音数据。
* **解耦发送架构**：采用同步 `mix_and_prepare` 与外部异步 `send_to` 的解耦模式。避免在 UDP 网络发送的 `.await` 期间持有会议锁，极大降低了高并发下的锁竞争与音频丢包率。
* **静音与旁路**：支持针对特定参会成员的静音和旁路控制。

### 3. 音频录音 (`recording`)
* **高可靠录音**：提供向本地文件系统直接写入标准 WAV（PCM 16-bit）格式的录音功能。
* **故障反馈机制**：录音 worker 的 I/O 错误（如磁盘已满、无写入权限）会通过 `RecordingSessionInfo` 上的原子变量 `has_error` 进行透传，防止写盘失败时控制面静默通过。
* **生命周期管理**：支持录音按需启动、停止及文件分片轮转。

### 4. 实时媒体转码 (`live_transcode`)
* **编解码支持**：实现 G.711 (PCMA/PCMU) 与 linear PCM 之间的编解码。
* **动态重采样**：支持 8kHz 与 16kHz/48kHz 音频流之间的动态重采样，为对接智能客服、实时 AI 语音流等高级业务提供底层支持。

### 5. DTMF 探测与生成 (`dtmf`)
* **RFC 2833 / Telephone-Event**：负责 RTP 带内 DTMF 信号的提取与合成。
* **带外转换**：提供 DTMF 从 RTP 事件到 SIP INFO 报文的转换中间层。

### 6. RTCP 与服务质量指标 (`rtcp`, `metrics`)
* **抖动缓冲**：提供抖动（Jitter）和丢包率（Packet Loss）的数学估算模型。
* **MOS 分估算**：根据时延、抖动、丢包率等因子，在线实时计算当前通话的 MOS (Mean Opinion Score) 语音质量分，用于 CDR 数据统计。

### 7. WAV 辅助解析 (`wav_reader`)
* **WAV 解析器**：用于加载和解析彩铃、IVR 语音导航提示音（如 `welcome.wav`）。

---

## 设计哲学

1. **高性能与低延迟**：由于媒体处理是 CPU 密集型任务，算法设计最大化减少堆内存分配，采用预分配缓冲区。
2. **高并发锁安全**：采用细粒度并发设计，遵循 VOS-RS 规范，严禁跨越 `.await` 边界持有任何标准互斥锁（`std::sync::Mutex`），确保高负载下通话媒体流不发生抖动或静音。
3. **零 panic 安全性**：作为电信级平台核心库，所有操作均返回 `Result` 错误，严禁在生产路径中使用 `unwrap` 或引发 panic。

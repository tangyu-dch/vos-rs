# 防盗打事件数据库记录

## 问题
当前 sip-edge 在防盗打检查失败时仅做 `warn!` 日志输出，不写入数据库。安全组无法追踪何时、何地、为何触发了防盗打规则。

## 方案
在 sip-edge 的防盗打检查失败时，将事件写入 `anti_fraud_events` 表。

### 修改文件

**1. `services/sip-edge/src/main.rs`** (行 3241-3252)
- 在 `check_call` 返回错误后，调用 `log_anti_fraud_event` 写入数据库
- sip-edge 已有 `db_store: Option<PostgresCdrStore>` 可获取 PgPool
- 需要新增辅助函数 `log_anti_fraud_event`

**2. `services/sip-edge/src/anti_fraud.rs`**
- 为 `AntiFraudError` 实现 `as_event_type()` 方法，返回对应的事件类型字符串（如 `blocked_ip`, `blocked_number`, `international_blocked` 等）

### 实现细节

在 `main.rs` 的防盗打检查失败分支中添加：

```rust
if let Err(e) = edge_state.anti_fraud_engine.check_call(&username, peer.ip(), &destination) {
    warn!(%username, %destination, error = %e, "anti-fraud check failed");
    
    // 记录防盗打事件到数据库
    if let Some(ref db) = edge_state.db_store {
        let pool = db.pool();
        let event_type = e.as_event_type();
        let source_ip = peer.ip().to_string();
        let detail = e.to_string();
        tokio::spawn(async move {
            let _ = sqlx::query(
                "INSERT INTO anti_fraud_events (event_type, source_ip, account, destination, detail) 
                 VALUES ($1, $2::inet, $3, $4, $5)"
            )
            .bind(&event_type)
            .bind(&source_ip)
            .bind(&username)
            .bind(&destination)
            .bind(&detail)
            .execute(pool)
            .await;
        });
    }
    
    return vec![PendingDatagram::new(...)];
}
```

在 `anti_fraud.rs` 添加：

```rust
impl AntiFraudError {
    pub fn as_event_type(&self) -> &'static str {
        match self {
            Self::IpBlocked => "blocked_ip",
            Self::IpNotAllowed => "ip_not_allowed",
            Self::BlockedNumber => "blocked_number",
            Self::InternationalBlocked => "international_blocked",
            Self::PremiumNumberBlocked => "premium_blocked",
            Self::ConcurrentLimitExceeded => "concurrent_exceeded",
            Self::IpConcurrentLimitExceeded => "ip_concurrent_exceeded",
            Self::CpsLimitExceeded => "cps_exceeded",
            Self::InsufficientBalance => "insufficient_balance",
            Self::ShortCallPatternDetected => "short_call_pattern",
        }
    }
}
```

### 验证
1. `cargo test -p sip-edge` 全部通过
2. 重启 sip-edge，用 SIPp 发送黑名单号码呼叫
3. 查询数据库：`SELECT * FROM anti_fraud_events ORDER BY created_at DESC LIMIT 10`
4. 确认事件被正确记录（event_type, source_ip, account, destination, detail）
